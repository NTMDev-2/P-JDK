# How Releases Work
Whenever I make a major or big change to the PJDK, like fixing some bugs or changing / creating a significant feature, I will bundle many (or 1) feature into a release and push it. 
Generally, a release with `x.x.0` signifies a very major release, adding very important features. A `x.x.5` is like a less-important release, but still added many notable features.
A `x.x.x.5` is a small release, and a `x.x.x` (not `.5 or `.0`) just means a smaller patch / update release.

# How to use
Open the "Assets" dropdown list, find the PJDK.py link, then click on it to download
Make sure you download from the latest release! The latest release will have a "Latest" tag next to it

# Issues
Here are what all the labels mean:
- Feature: This "item" or "code" is not present in the PJDK, but doesn't necessarily break it, but rather a missing "item" or "code" that should be added to make PJDK better
- Major Issue: Something very important is broken, and must be fixed ASAP
- No Development: This is not being currently worked on, and may not be worked on for many updates.
- Not Essential: Just a small typo or little nitpicky issue; making people's lives easier
- Postponed: This is a less harsh version of `No Development`
- Priority: This issue will come out in the current version of PJDK (if I get to it in time)
- Quick Fix: Can be fixed in like 2 seconds
- Upcoming Fix: Usually, when an issue has this tag, it is either already fixed and planned on to be included in the next update, or will is being fixed and will come out in the next update
